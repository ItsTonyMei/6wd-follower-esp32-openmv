# ROADMAP — 履带车视觉跟随系统

## 架构总览

```
┌──────────────────────────────────────────────────────────────┐
│                     L1: 感知层 (OpenMV)                       │
│   YOLO person detection → 多特征融合距离估计 → VIS/UART → ESP32 │
├──────────────────────────────────────────────────────────────┤
│                     L2: 决策层 (ESP32)                        │
│   FollowLogic + 行为仲裁 + WiFi Dashboard + 远程调参          │
│   接收: OpenMV VIS帧 + 遥控SBUS + STM32遥测                   │
│   输出: MotorCmd → UART → STM32                              │
├──────────────────────────────────────────────────────────────┤
│                   L3: 执行与安全层 (STM32)                     │
│   PWM电机驱动 + 编码器PID + 命令超时停车 + 硬线急停 + 过流保护  │
│   独立于L1/L2的安全链路，任何上层故障均可自主切断动力            │
└──────────────────────────────────────────────────────────────┘
```

## 安全链路设计（5条独立关断路径）

| # | 触发源 | 机制 | 响应时间 |
|---|--------|------|---------|
| 1 | 人按急停按钮 | 继电器物理断开电机电源 | <10ms |
| 2 | STM32 检测过流 | ADC阈值 → PWM归零 + 错误码上报 | <1ms |
| 3 | STM32 命令超时 | 500ms无有效命令 → 自动STOP | <500ms |
| 4 | ESP32 遥控失联 | SBUS failsafe flag → STOP | <100ms |
| 5 | ESP32 视觉超时 | VISION_TIMEOUT_MS → 降级STOP或切Manual | <700ms |

## 数据流

```
OpenMV                        ESP32                          STM32
───────                      ───────                        ───────
YOLO检测                      VisionBridge.parse()           UART1 RX中断
  │                             │                              │
  ▼                             ▼                              ▼
距离估计(0-1)                 VisState更新                   MotorCmd解析
  │                             │                           (CRC8校验)
  ▼                             ▼                              │
VIS帧组装                     FollowLogic.update()             ▼
  │                           · 视觉距离保持                  电机PWM输出
  ▼                           · 超声波避障                     │
UART3.write() ──────────────► · 行为仲裁                      ▼
                               │                           编码器ADC采集
                               ▼                              │
                             MotorCmd + CRC8 ──────────────►  ▼
                                                            遥测帧组装
                               ◄──────────────────────  (编码器/电流/错误)
                               │
                               ▼
                             DataAggregator
                               │
                               ▼
                             Dashboard WebSocket (WiFi)
```

## Phase 1 — STM32 底层驱动与安全层 (预计 3-5 天)

**目标**: STM32 可以独立控制履带电机，安全机制全部生效。本阶段不依赖 ESP32 和 OpenMV。

### 1.1 PWM 电机驱动 + 方向控制
- [ ] 配置 TIM1 4路 PWM（CH1/CH2=左履带前后, CH3/CH4=右履带前后）
- [ ] 实现 `void motorWrite(CarCmd cmd, uint8_t pwm)` 函数
- [ ] 方向切换时插入 MOTOR_DEAD_TIME_MS 死区
- [ ] 用示波器验证 PWM 波形正确
- [ ] 电机空载测试：FWD/REV/LFT/RGT/STOP 各方向正确

### 1.2 UART 命令接收与解析
- [ ] USART1 中断接收，环形缓冲区
- [ ] 定长二进制协议：`[0xAA][CMD:1B][PWM:1B][CRC8:1B]` 共 4 字节
- [ ] 帧头 0xAA 同步 + CRC8 校验
- [ ] 解析后更新内部 `MotorCmd latestCmd`

### 1.3 命令超时自动停车
- [ ] SysTick 1ms 中断驱动超时计数器
- [ ] 收到有效命令 → 清零计数器
- [ ] 超时 500ms → `motorWrite(STOP, 0)` + 置位超时标志
- [ ] 超时后需收到连续 3 个有效帧才能恢复

### 1.4 硬线急停逻辑
- [ ] GPIO 输入检测急停按钮状态（常闭触点，断开=急停）
- [ ] GPIO 输出驱动继电器（常开，失电=断开电机电源）
- [ ] 急停触发 → 继电器断开 + PWM 归零 + 上报错误码
- [ ] 恢复流程：急停按钮复位 + ESP32 发送解除命令（或物理钥匙开关）

### 1.5 编码器脉冲计数 (可选)
- [ ] TIM2/TIM3 编码器模式配置
- [ ] 速度计算（每 10ms 的脉冲增量 → RPM）
- [ ] 若无编码器则跳过，后续补装

### 1.6 电流采样与过流保护
- [ ] ADC1 双通道采样左右电机电流（ACS712 或 shunt 电阻）
- [ ] 过流阈值：额定 2x 持续 100ms 或 额定 5x 瞬时 → 关断
- [ ] 过流触发 → PWM 归零 + 错误码 CURRENT_FAULT 上报

## Phase 2 — 遥控器集成 (预计 2-3 天)

**目标**: 遥控器可以开车，手动模式可用。此阶段不需要 OpenMV 视觉。

### 2.1 SBUS 接收机协议解析
- [ ] ESP32 UART2 接 SBUS 接收机（100kbps, 8E2, 反相需硬件反相器或软件反相）
- [ ] 解析 25 字节帧：16 通道 × 11-bit（172-1811 → 0-100%）
- [ ] 通道映射：CH1(右摇杆X)=转向, CH2(左摇杆Y)=油门, CH5=模式开关
- [ ] 验证各通道值范围正确

### 2.2 遥控器 → 电机直通控制
- [ ] 摇杆映射为 MotorCmd（油门 → FWD/REV + pwm 值，转向 → 差速比例）
- [ ] 差速映射公式：`pwm_L = throttle ± steer_factor`, `pwm_R = throttle ∓ steer_factor`
- [ ] MotorCmd 通过 UART 发送到 STM32
- [ ] 实际开起来验证操控手感

### 2.3 遥控失联 FailSafe
- [ ] SBUS 帧 failsafe bit 检测
- [ ] 失联 → 立即发送 STOP
- [ ] 恢复 → 需先回中油门 + 再操作才解锁（防止飞车）

### 2.4 模式切换 Manual / Auto / STOP
- [ ] CH5 三档开关读取：
  - 位置1: Manual Mode（遥控驾驶）
  - 位置2: Auto-Follow Mode（视觉跟随）
  - 位置3: Emergency Stop（强制停车）
- [ ] 模式切换时平滑过渡（PWM 渐变而非突变）

## Phase 3 — ESP32 ↔ STM32 UART 通信协议 (预计 2-3 天)

**目标**: 双板双向通信稳定可靠，有 CRC、有超时、有心跳。

### 3.1 双向二进制协议设计
```
下行 (ESP32 → STM32, 每 50ms):
  [0xAA] [CMD:1B] [PWM:1B] [CRC8:1B]
  共 4 字节

上行 (STM32 → ESP32, 收到命令后立即回传, 或每 100ms 主动上报):
  [0xBB] [EncL:2B] [EncR:2B] [CurL:1B] [CurR:1B] [Flags:1B] [ErrCode:1B] [CRC8:1B]
  共 10 字节

Flags bit定义:
  bit0: 急停状态
  bit1: 命令超时
  bit2: 过流警告
  bit3: 心跳
ErrCode 错误码:
  0x00: 正常
  0x01: 过流关断
  0x02: 急停触发
  0x03: 命令超时
  0x04: 编码器故障
```

### 3.2 ESP32 端 UART 发送 + 接收
- [ ] USART2 → STM32 发送 MotorCmd（Arduino `Serial2`）
- [ ] USART2 中断接收遥测帧
- [ ] DataAggregator 新增字段：encL, encR, curL, curR, stmFlags, stmErrCode

### 3.3 STM32 端遥测回传
- [ ] 收到下行帧 → 回传当前状态
- [ ] 无下行帧时每 100ms 主动发送心跳
- [ ] 遥测帧同样带 CRC8

### 3.4 心跳 + 通信异常处理
- [ ] ESP32 端检测遥测超时（300ms 无回传 → Dashboard 报警）
- [ ] STM32 端命令超时（500ms → STOP，延续 1.3）
- [ ] UART 帧错误率超过 10% → Dashboard 报警

## Phase 4 — ESP32 主决策层 (预计 3-5 天)

**目标**: 视觉跟随功能完整可用，距离保持逻辑正确，行为仲裁优先级清晰。

### 4.1 剥离现有代码
- [ ] `MotorTask.cpp` → 删除 PWM 直驱，改为 UART MotorCmd 发送
- [ ] `DataAggregator.h` → 新增 STM32 遥测字段
- [ ] `Tasks.cpp` → 调整任务结构适应新架构
- [ ] 保留但可能移除超声波避障（履带车宽体和重量下超声波实用性存疑，后续评估）

### 4.2 FollowLogic 重构: 差速转向
- [ ] 输入：VisState（cx偏航, distScore距离）, RC 通道值
- [ ] 输出：MotorCmd（CarCmd + pwm）
- [ ] 转向控制：`pwm_diff = Kp * (cx - FRAME_CX) / FRAME_CX * MAX_PWM`
- [ ] 差速输出：`pwm_L = base_pwm - pwm_diff`, `pwm_R = base_pwm + pwm_diff`
- [ ] 限幅保护：pwm 不超 PWM_RANGE

### 4.3 VisionBridge 集成
- [ ] 确保 VIS 帧正常进入 VisState
- [ ] 合理性校验：w∈[10,320], h∈[10,240], conf∈[0,1]
- [ ] 异常数据过滤 + 计数上报

### 4.4 行为仲裁器
```
优先级链（从高到低）:
1. 急停 (STM32 硬线 + 遥控CH5=3)
2. STM32 错误码 (过流/编码器故障)
3. 遥控失联
4. 命令超时 (STM32 端已处理，ESP32 不重复发)
5. 视觉超时 (VIS 帧 > VISION_TIMEOUT_MS 未更新)
6. 遥控 Manual Mode
7. 视觉跟随 Auto-Follow Mode
8. 默认: STOP
```

### 4.5 距离保持逻辑
| distScore 范围 | 行为 | 命令 |
|----------------|------|------|
| < 0.25 (太远) | 全速前进追赶 | FWD, pwm=MAX |
| 0.25-0.45 (较远) | 中速跟进 | FWD, pwm=120 |
| 0.45-0.60 (合适) | 微调保持 | FWD/STOP 交替 |
| 0.60-0.80 (较近) | 慢速后退 | REV, pwm=80 |
| > 0.80 (太近) | 全速后退 | REV, pwm=MAX |

滞后设计（防振荡）：
- 进入"合适"区后需偏离 ±0.05 才触发移动
- 状态转换需维持 3 帧（~600ms）确认

## Phase 7 — 全系统联调 (预计 3-5 天)

**目标**: 三板协同工作，端到端链路打通，PID 整定完成，安全测试全覆盖。

### 7.1 上电启动顺序
- [ ] 启动流程：STM32 先上电自检（PWM 锁定 0）→ ESP32 上电连接 → 握手 → 解锁
- [ ] 握手协议：ESP32 发 `HANDSHAKE_REQ` → STM32 回 `HANDSHAKE_ACK` + 版本号
- [ ] OpenMV 最后手动上电（或由 ESP32 GPIO 控制上电），启动慢不影响安全

### 7.2 端到端链路验证
- [ ] 人站在摄像头前 → 确认 OpenMV 检测 → ESP32 决策 → STM32 驱动
- [ ] 全链路延迟测试：人在摄像头前移动 → 车轮响应时间 < 300ms 为合格
- [ ] 静态跟随测试：人保持不动，车保持距离稳定（不前后振荡）

### 7.3 PID 整定
- [ ] STM32 速度环 PID：P（比例增益）→ I（消除静差）→ D（抑制超调）
- [ ] ESP32 FollowLogic 转向 P 控制器（纯比例通常够用）
- [ ] 记录最优参数到 Config.h + NVS

### 7.4 安全机制测试清单
| 测试项 | 操作 | 预期结果 |
|--------|------|---------|
| 急停按钮 | 运行中按下 | 继电器断开，车立即停 |
| 过流保护 | 堵转电机 | <1s 内 PWM 归零 |
| 命令超时 | 拔掉 ESP32-STM32 UART 线 | <500ms 车自动停 |
| 遥控失联 | 运行中关遥控器 | <100ms 车自动停 |
| 视觉超时 | 运行中遮住摄像头 | <700ms 车自动停 |
