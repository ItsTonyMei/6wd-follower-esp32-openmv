# ROADMAP — 履带车视觉跟随系统

## 架构总览

```
┌──────────────────────────────────────────────────────────────┐
│                     L1: 感知层 (OpenMV)                       │
│   YOLO person detection → 多特征融合距离估计 → VIS/UART → ESP32 │
├──────────────────────────────────────────────────────────────┤
│                     L2: 决策层 (ESP32)                        │
│   FollowLogic + 行为仲裁 + WiFi Dashboard + 远程调参          │
│   接收: OpenMV VIS帧 + 遥控SBUS + STM32遥测                   │
│   输出: MotorCmd → UART → STM32                              │
├──────────────────────────────────────────────────────────────┤
│                   L3: 执行与安全层 (STM32)                     │
│   PWM电机驱动 + 编码器PID + 命令超时停车 + 硬线急停 + 过流保护  │
│   独立于L1/L2的安全链路，任何上层故障均可自主切断动力            │
└──────────────────────────────────────────────────────────────┘
```

## 安全链路设计（5条独立关断路径）

| # | 触发源 | 机制 | 响应时间 |
|---|--------|------|---------|
| 1 | 人按急停按钮 | 继电器物理断开电机电源 | <10ms |
| 2 | STM32 检测过流 | ADC阈值 → PWM归零 + 错误码上报 | <1ms |
| 3 | STM32 命令超时 | 500ms无有效命令 → 自动STOP | <500ms |
| 4 | ESP32 遥控失联 | SBUS failsafe flag → STOP | <100ms |
| 5 | ESP32 视觉超时 | VISION_TIMEOUT_MS → 降级STOP或切Manual | <700ms |

```
───────                      ───────                        ───────

  │                             │                              │

  │                             │                           (CRC8校验)

                               │

                               ▼

```

## Phase 1 — STM32 底层驱动与安全层 (预计 3-5 天)

**目标**: STM32 可以独立控制履带电机，安全机制全部生效。本阶段不依赖 ESP32 和 OpenMV。

### 1.1 RC PWM 电调控制
- [ ] **先确认电调控制接口协议**（查阅电调说明书：标准 RC PWM 50Hz / UART / CAN）
- [ ] 假设标准 RC PWM：中位 1500μs = STOP, 1500-2000μs = 前进, 1000-1500μs = 后退
- [ ] 配置 TIM1 CH1/CH2 输出 50Hz 标准舵机 PWM 信号到电调左右通道
- [ ] 实现 `void escWrite(CarCmd cmd, uint8_t pwm)` — 将 CarCmd + pwm 映射为 μs 脉宽
- [ ] 映射公式: FWD → `1500 + map(pwm, 0,255, 0,500)` μs; REV → `1500 - map(pwm, 0,255, 0,500)` μs
- [ ] 用示波器或舵机测试器验证 PWM 脉宽正确（中位 ±5μs 精度）
- [ ] 电调上电自检流程：上电时输出中位 STOP，等待电调初始化完成（听蜂鸣声）
- [ ] 履带空载测试：FWD / REV / 原地差速转向 / STOP 各方向正确
- [ ] **注意**: 电调内部已处理 H-bridge 换向和死区，STM32 无需再做死区逻辑

### 1.2 UART 命令接收与解析
- [ ] USART1 中断接收，环形缓冲区
- [ ] 定长二进制协议：`[0xAA][CMD:1B][PWM:1B][CRC8:1B]` 共 4 字节
- [ ] 帧头 0xAA 同步 + CRC8 校验
- [ ] 解析后更新内部 `MotorCmd latestCmd`

### 1.3 命令超时自动停车
- [ ] SysTick 1ms 中断驱动超时计数器
- [ ] 收到有效命令 → 清零计数器
- [ ] 超时 500ms → `motorWrite(STOP, 0)` + 置位超时标志
- [ ] 超时后需收到连续 3 个有效帧才能恢复

### 1.4 硬线急停逻辑
- [ ] GPIO 输入检测急停按钮状态（常闭触点, 断开=急停）
- [ ] GPIO 输出驱动 48V 大电流继电器（常开, 失电=断开电调动力电）
- [ ] 急停回路设计：电池+ → 继电器常开触点 → 电调 48V 输入。继电器线圈由 STM32 GPIO + 急停按钮串联控制
- [ ] 急停触发 → 继电器断开 + PWM 归零 + 上报错误码
- [ ] 恢复流程：急停按钮复位 + ESP32 发送解除命令（或物理钥匙开关）
- [ ] **安全设计**: 急停链路为常闭逻辑 — 断线/断电/STM32死机 任一故障均触发急停

### 1.5 编码器脉冲计数 (可选)
- [ ] TIM2/TIM3 编码器模式配置
- [ ] 速度计算（每 10ms 的脉冲增量 → RPM）
- [ ] 若无编码器则跳过，后续补装

### 1.6 电流采样与过流保护
- [ ] **先确认电调是否有电流回传功能**（UART/CAN 回传或模拟电压输出）
- [ ] 若电调无回传: ADC 采样左右电机电流（ACS712 串联在电调输出到电机之间）
- [ ] 过流阈值设定（根据电机额定电流 ×1.5 持续, ×3 瞬时）
- [ ] 过流触发 → PWM 归零 + 错误码 CURRENT_FAULT 上报 + 可选继电器断开
- [ ] 若电调自带过流保护: STM32 只需监控电调回传的状态字

## Phase 2 — 遥控器集成 (预计 3-4 天)

**目标**: ELRS/CRSF 遥控链路打通，Manual 模式可用。此阶段不需要 OpenMV 视觉。

### 遥控设备选型
| 设备 | 推荐型号 | 说明 |
|------|---------|------|
| 遥控器 | **Jumper T14 ELRS** | EdgeTX 系统, 内置 ELRS 2.4G 1W, Hall 摇杆, USB-C 充电 |
| 接收机 | **Radiomaster ER6** / **Happymodel EP1** | ELRS 2.4G, CRSF 输出, 双向遥测 |
| 备选遥控 | Radiomaster Boxer / TX16S | 同样 EdgeTX + ELRS, 屏幕更大 |
| 协议 | **CRSF** (Crossfire) | 420k baud, 16ch × 11-bit, 双向遥测 |

> Jumper T14: EdgeTX 系统, 内置 ELRS 模块最高 1W 发射功率, 开源固件可深度定制混控和逻辑开关。接收机通过 UART 输出 CRSF 协议帧到 ESP32。

### 2.1 ELRS 接收机绑定与 CRSF 库集成
- [ ] UART1 (RX=GPIO15, TX=GPIO4) 接 ELRS 接收机 CRSF 端口（TX↔RX 交叉）
- [ ] 接收机供电：5V（从 48V→5V 降压器取电）
- [ ] 安装 [CRSFforArduino](https://github.com/ZZ-Cat/CRSFforArduino) 库
- [ ] ELRS 发射端和接收机刷相同版本固件，完成 Bind
- [ ] 运行示例代码验证 16 通道数据流正常接收
- [ ] 关键设置: EdgeTX 内部模块 baud = 921k（EdgeTX 默认即可）；ELRS Packet Rate = 250Hz（地面车辆不需要 FPV 无人机的高刷新率）

### 2.2 通道映射与遥控器配置
- [ ] EdgeTX Mixer 配置：
  - **CH1**: 右摇杆 X 轴 → Steering（转向，自回中）
  - **CH2**: 备用（可映射摄像头俯仰或暂留空）
  - **CH3**: 左摇杆 Y 轴 → Throttle（油门，建议设回中弹簧 or 取消回中用棘齿手感）
  - **CH4**: 备用
  - **CH5**: 三档开关 SWA → Mode: Manual / Auto-Follow / Emergency Stop
  - **CH6**: 旋钮 POT1 → Max Speed 限速调节
- [ ] CRSF 通道值范围: 172-1811（中位 992），映射为 -100% ~ 0 ~ +100%
- [ ] 在 ESP32 端实现 `CRSFReader` 类，封装通道值读取

### 2.3 遥控器 → 电机直通控制 (Manual Mode)
- [ ] CH3 油门 → FWD/REV + pwm 值: 中位以上 = 前进, 中位以下 = 后退
- [ ] CH1 转向 → 差速比例: 摇杆左 → 左履带减速/右履带加速, 摇杆右 → 反之
- [ ] 差速映射: `pwm_L = throttle - steer_factor`, `pwm_R = throttle + steer_factor`
- [ ] CH6 旋钮限速: pwm 上限 = map(pot1, 0~100%, MIN_PWM~MAX_PWM)
- [ ] 死区设置: 摇杆中位 ±3% 视为 STOP（避免摇杆抖动导致车微动）
- [ ] MotorCmd 通过 UART2 发送到 STM32
- [ ] 实际开起来验证操控手感和履带差速比例

### 2.4 遥控失联 FailSafe
- [ ] CRSF 协议自带链路质量监控: RSSI (dBm) + LQ (Link Quality %)
- [ ] LQ < 50% 持续 300ms → Dashboard 低信号警告
- [ ] LQ = 0% / 接收机完全失联 → ELRS 接收机内部自动进入 FailSafe（所有通道输出预设的 failsafe 值）
- [ ] EdgeTX 端配置 FailSafe 位置: CH3 回中（=0%油门）, CH5 切到 Emergency Stop
- [ ] ESP32 端额外检测: CRSF 帧中断超过 100ms → 强制 STOP
- [ ] 恢复：信号恢复后需 CH3 回中 + CH5 切回 Manual 才解锁

### 2.5 模式切换 Manual / Auto / STOP
- [ ] CH5 (SWA 三档) 读取: 上=Manual(遥控驾驶), 中=Auto-Follow(视觉跟随), 下=E-Stop(紧急停车)
- [ ] 模式切换时 ESP32 行为仲裁器收到模式变更事件
- [ ] Auto→Manual 切换: 立即切换到遥控驾驶, 无延迟
- [ ] Manual→Auto 切换: 先 STOP 100ms, 确认视觉数据有效后才切入跟随
- [ ] 任何模式 → E-Stop: 立即 STOP + 持续发送 STOP 命令

### 2.6 CRSF 双向遥测（可选增强）
- [ ] ESP32 → ELRS 接收机回传: 电池电压、车速、模式、错误码
- [ ] Jumper T14 屏幕上实时显示遥测数据（通过 EdgeTX Telemetry 页面配置）
- [ ] 利用 EdgeTX Logical Switches 设置: 电压过低振动报警、错误码触发语音提示

## Phase 3 — ESP32 ↔ STM32 UART 通信协议 (预计 2-3 天)

**目标**: 双板双向通信稳定可靠，有 CRC、有超时、有心跳。

### 3.1 双向二进制协议设计
```
下行 (ESP32 → STM32, 每 50ms):
  [0xAA] [CMD:1B] [PWM:1B] [CRC8:1B]
  共 4 字节

上行 (STM32 → ESP32, 收到命令后立即回传, 或每 100ms 主动上报):
  [0xBB] [EncL:2B] [EncR:2B] [CurL:1B] [CurR:1B] [Flags:1B] [ErrCode:1B] [CRC8:1B]
  共 10 字节

Flags bit定义:
  bit0: 急停状态
  bit1: 命令超时
  bit2: 过流警告
  bit3: 心跳
ErrCode 错误码:
  0x00: 正常
  0x01: 过流关断
  0x02: 急停触发
  0x03: 命令超时
  0x04: 编码器故障
```

### 3.2 ESP32 端 UART 发送 + 接收
- [ ] USART2 → STM32 发送 MotorCmd（Arduino `Serial2`）
- [ ] USART2 中断接收遥测帧
- [ ] DataAggregator 新增字段：encL, encR, curL, curR, stmFlags, stmErrCode

### 3.3 STM32 端遥测回传
- [ ] 收到下行帧 → 回传当前状态
- [ ] 无下行帧时每 100ms 主动发送心跳
- [ ] 遥测帧同样带 CRC8

### 3.4 心跳 + 通信异常处理
- [ ] ESP32 端检测遥测超时（300ms 无回传 → Dashboard 报警）
- [ ] STM32 端命令超时（500ms → STOP，延续 1.3）
- [ ] UART 帧错误率超过 10% → Dashboard 报警

## Phase 4 — ESP32 主决策层 (预计 3-5 天)

**目标**: 视觉跟随功能完整可用，距离保持逻辑正确，行为仲裁优先级清晰。

### 4.1 剥离现有代码
- [ ] `MotorTask.cpp` → 删除 PWM 直驱，改为 UART MotorCmd 发送
- [ ] `DataAggregator.h` → 新增 STM32 遥测字段
- [ ] `Tasks.cpp` → 调整任务结构适应新架构
- [ ] 保留但可能移除超声波避障（履带车宽体和重量下超声波实用性存疑，后续评估）

### 4.2 FollowLogic 重构: 差速转向
- [ ] 输入：VisState（cx偏航, distScore距离）, RC 通道值
- [ ] 输出：MotorCmd（CarCmd + pwm）
- [ ] 转向控制：`pwm_diff = Kp * (cx - FRAME_CX) / FRAME_CX * MAX_PWM`
- [ ] 差速输出：`pwm_L = base_pwm - pwm_diff`, `pwm_R = base_pwm + pwm_diff`
- [ ] 限幅保护：pwm 不超 PWM_RANGE

### 4.3 VisionBridge 集成
- [ ] 确保 VIS 帧正常进入 VisState
- [ ] 合理性校验：w∈[10,320], h∈[10,240], conf∈[0,1]
- [ ] 异常数据过滤 + 计数上报

### 4.4 行为仲裁器
```
优先级链（从高到低）:
1. 急停 (STM32 硬线 + 遥控CH5=3)
2. STM32 错误码 (过流/编码器故障)
3. 遥控失联
4. 命令超时 (STM32 端已处理，ESP32 不重复发)
5. 视觉超时 (VIS 帧 > VISION_TIMEOUT_MS 未更新)
6. 遥控 Manual Mode
7. 视觉跟随 Auto-Follow Mode
8. 默认: STOP
```

### 4.5 距离保持逻辑
| distScore 范围 | 行为 | 命令 |
|----------------|------|------|
| < 0.25 (太远) | 全速前进追赶 | FWD, pwm=MAX |
| 0.25-0.45 (较远) | 中速跟进 | FWD, pwm=120 |
| 0.45-0.60 (合适) | 微调保持 | FWD/STOP 交替 |
| 0.60-0.80 (较近) | 慢速后退 | REV, pwm=80 |
| > 0.80 (太近) | 全速后退 | REV, pwm=MAX |

滞后设计（防振荡）：
- 进入"合适"区后需偏离 ±0.05 才触发移动

## Phase 7 — 全系统联调 (预计 3-5 天)

**目标**: 三板协同工作，端到端链路打通，PID 整定完成，安全测试全覆盖。

### 7.1 上电启动顺序
- [ ] 启动流程：STM32 先上电自检（PWM 锁定 0）→ ESP32 上电连接 → 握手 → 解锁
- [ ] 握手协议：ESP32 发 `HANDSHAKE_REQ` → STM32 回 `HANDSHAKE_ACK` + 版本号
- [ ] OpenMV 最后手动上电（或由 ESP32 GPIO 控制上电），启动慢不影响安全

### 7.2 端到端链路验证
- [ ] 人站在摄像头前 → 确认 OpenMV 检测 → ESP32 决策 → STM32 驱动
- [ ] 全链路延迟测试：人在摄像头前移动 → 车轮响应时间 < 300ms 为合格
- [ ] 静态跟随测试：人保持不动，车保持距离稳定（不前后振荡）

### 7.3 PID 整定
- [ ] STM32 速度环 PID：P（比例增益）→ I（消除静差）→ D（抑制超调）
- [ ] ESP32 FollowLogic 转向 P 控制器（纯比例通常够用）
- [ ] 记录最优参数到 Config.h + NVS

### 7.4 安全机制测试清单
| 测试项 | 操作 | 预期结果 |
|--------|------|---------|
| 急停按钮 | 运行中按下 | 继电器断开，车立即停 |
| 过流保护 | 堵转电机 | <1s 内 PWM 归零 |
| 命令超时 | 拔掉 ESP32-STM32 UART 线 | <500ms 车自动停 |
| 遥控失联 | 运行中关遥控器 | <100ms 车自动停 |
| 视觉超时 | 运行中遮住摄像头 | <700ms 车自动停 |
| 遥控恢复 | 重新开遥控器 | 需手动回中后解锁 |

## Phase 8 — 场地测试与迭代 (预计 5-7 天)

**目标**: 真实环境验证系统可靠性和鲁棒性。

### 8.1 室内平地跟随
- [ ] 光滑地面直线跟随（1m/s 慢速）
- [ ] 90° 转弯跟随（人转弯车能否跟上）
- [ ] 后退保持测试（人走向车）
- [ ] 记录指标：跟踪丢失率、距离保持误差

### 8.2 户外/坡道测试
- [ ] 水泥路面 → 草地 → 砂石路面
- [ ] 10° 缓坡启停（车能否在坡道上静止不滑坡）
- [ ] 颠簸路面视觉稳定性（摄像头是否抖到丢失目标）

### 8.3 紧急场景测试
- [ ] 人突然从侧面闯入（检测响应时间）
- [ ] 目标突然消失（车是否平稳停止）
- [ ] 电机堵转（过流保护响应）
- [ ] 通信线缆松脱（系统是否安全停车而非失控）

### 8.4 续航与散热
- [ ] 满载等效负载连续运行（2×500W 峰值 → 48V 89Ah 理论续航约 4 小时 @ 50% 负载）
- [ ] 电池电压从满电 (~54.6V) 到截止 (~36V)，记录实际运行时间
- [ ] 电调散热片温度（红外测温枪，持续满负载工况）
- [ ] 48V→5V 降压模块温度（10A 额定，实际控制电路 <3A，余量充足）
- [ ] STM32/ESP32 芯片温度
- [ ] 若过热 → 加散热片/风扇（电调位置需通风）
